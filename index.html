HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The footer element contains the author, copyright, contact, sitemap, and navigation.
  • The style element defines the CSS styles associated with a website.
  • The title element defines the title for a webpage.
  • The base element defines the base URL for a webpage.
  • The link element connects an external resource to the HTML document.
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport.
  • The script element embeds client-side scripts in the HTML document.
  • The h1, h2, h3, h4, h5, and h6 elements represent the level of heading a given text block represents.
  • The p element represents a paragraph or block of text.
  • The ul, ol, and li elements represent unordered lists, ordered lists, and list items.
  • The img element contains information about images that are displayed on the webpage.
  • The br element creates a line of empty space, or a line break, between two blocks of content.
  • The a element (which stands for anchor) creates links to the same webpage or other webpages.

CSS

  • The selector points to the HTML element where the CSS style is to be applied.
  • The period preceding the selector designates this as a class selector. Any HTML element assigned to the class, card, will gain these property assignments from the CSS.
  • A declaration is a combination of a property and its corresponding value.
  • The property is the part of a declaration that defines the specific aspect of an element you want to modify.
  • The value is the part of the declaration that sets the parameter for a given property.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git status: checks what branch we are currently on
  • git checkout: -b branch-name: creates a new branch and switches to it
  • git pull origin (target): pulls down latest target branch (main)
  • git checkout -b (name): creates a new branch of the main branch
  • git add -A: use to stage and commit all files on the working branch
  • git commit -m (“message”): commits files to the repository with a short message
  • git push origin (working branch): pushes working branch to GitHub

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
